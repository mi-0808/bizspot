# ワーキングスペース・サーチング「BisSpa（ビジスペ）」企画書

---

## 1. エグゼクティブサマリー

| 項目 | 内容 |
|------|------|
| プロジェクト名 | ワーキングスペース・サーチング「BisSpa（ビジスペ）」 |
| バージョン | v1.0 |
| 作成日 | 2026/3/27 |
| 作成者 | 松井湊 |
| ステータス | 企画中 |

**概要**

本アプリはフリーランスを対象に、作業やミーティングができる場所を探せる地図機能を提供する。
ユーザーが直面する「いい作業場を探したいが探すのが大変」という課題を解決し、より良い作業場に出会える体験の価値を届ける。

---

## 2. 背景・課題

### 2.1 市場背景
- テレワーク、フリーランス、起業文化の普及により、オフィス以外で仕事をする人が増えた
- オフィスを持たない人が作業場として自宅以外の良い場所を日々探している

### 2.2 ユーザーが抱える課題
- どこで作業すればいいか分からない（フリーランス初心者・学生）
- 普段使っている作業場に不満がある（騒がしい・Wi-Fiが弱い・長居しづらいなど）
- 出張先・外出先で突発的に作業場所が必要になる（ビジネスパーソン・経営者）

### 2.3 既存の代替手段と限界
| 代替手段 | 限界・不満点 |
|----------|-------------|
| 人に聞く | 情報のムラ、時間コスト |
| 飲食店を検索する | 作業向けの情報（電源・Wi-Fi・混雑度など）が載っていない |
| レンタルスペースを借りる | 費用が高い、短時間利用に不向きなケースが多い |

---

## 3. ターゲットユーザー

### 3.1 プライマリターゲット
- **ペルソナ名**：最近独立したフリーランス
- **年齢層・職種**：30代・Webデザイナー／ライター／ITフリーランスなど
- **主な利用シーン**：自宅以外での作業場所を朝・昼に探す場面、外出先でのスキマ時間

### 3.2 セカンダリターゲット
- **通信制高校生・大学生**：学校以外の場所で勉強・課題作業を行う必要がある層
- **中小企業の経営者・ビジネスパーソン**：出張・外回りが多く、オフィス外での作業機会が多い層

---

## 4. プロダクトコンセプト

### 4.1 コアバリュープロポジション
> **「自由に働くすべての人が、自分に合った場所を素早く見つけ、快適に仕事できるようになるアプリ」**

### 4.2 差別化ポイント
1. **集中できる度を確認できる**：混雑度・静粛性・Wi-Fi品質などのスコアを提供
2. **用途によって検索できる**：もくもく作業・ミーティング・短時間利用など目的別フィルター
3. **場所の種類が一目で分かるカラーピン**：マップ上のピンをスペースカテゴリ別に色分けし、地図を見ただけで何のお店かが分かる
4. **予約まで完結できる**：アプリ内から直接スペースの予約・確認ができる

## 5. 機能要件

### 5.1 MVP（最小実用製品）機能

| 優先度 | 機能名 | 概要 | 対象ユーザー |
|--------|--------|------|-------------|
| 必須 | マップ検索 | 現在地から近くのスペース（仕事ができる飲食店／コワーキングスペース／無料開放スペース／レンタルスペース）をカラーピンで表示する | 全員 |
| 必須 | カラーピン表示 | スペースのカテゴリ別にピンの色を分けて表示する（詳細は6.3参照） | 全員 |
| 必須 | 条件検索 | 用途・利用シーン・時間・料金などからスペースを絞り込んで検索する | 全員 |
| 推奨 | レビュー確認 | GoogleマップやシェアスペースのAPIからレビューや環境情報（電源・Wi-Fiなど）を確認できる | 初心者 |
| 推奨 | 履歴保存 | 過去に行ったスペースの滞在時間や作業内容を記録できる | ヘビーユーザー |

### 5.2 将来的な拡張機能（v2以降）
- コミュニティ機能（同じスペースを使うユーザー同士の交流）
- プロフィール機能（作業スタイルや好みに基づくパーソナライズ）
- イベント・勉強会の掲載機能

## 6. スペース情報ロジック（コア機能詳細）

### 6.1 スペース評価軸（案）
| 評価軸 | 説明 |
|--------|------|
| 静粛性 | 会話・BGM・騒音の少なさ |
| Wi-Fi品質 | 速度・安定性 |
| 電源の有無 | 座席数に対する電源の割合 |
| 混雑度 | 時間帯別の混みやすさ（リアルタイム or 統計ベース） |
| 料金 | 無料〜有料の範囲と料金体系 |
| 長居しやすさ | 最低消費額・滞在制限の有無 |

### 6.2 カラーピン設計（案）

マップ上のピンをスペースカテゴリ別に色分けし、一覧性・直感的な識別を実現する。

| カテゴリ | ピンカラー | 対象施設の例 |
|----------|-----------|-------------|
| カフェ | 🟡 黄色 | スターバックス・ドトール・個人経営カフェ など |
| ファストフード | 🔴 赤色 | マクドナルド・モスバーガー・吉野家 など |
| レンタルスペース | 🟢 緑色 | インスタベース掲載スペース・貸し会議室 など |
| コワーキングスペース | ⚫ 灰色 | WeWork・ESPACE・地域コワーキング など |
| 図書館・公共施設 | 🔵 青色 | 市立図書館・区民センター・自習室 など |

> **補足**：カテゴリが複合している施設（例：カフェ兼コワーキング）は、主機能に基づいて分類するか、将来的にはマルチタグ対応を検討する。

### 6.3 情報ソース（案）
- Google Maps API（場所情報・営業時間・レビュー）
- シェアスペース系API（ShareWis・インスタベース など）
- ユーザー投稿データ（UGCとして蓄積）

## 7. UX・画面構成（概要）

```
[ ホーム画面 ]
  ├── マップ（現在地表示 + 周辺スペースのピン）
  └── メニューバー
      ├── 条件で探す
      └── 保存済みスペース

[ マップ ]
  Step 1: 現在地を表示する
  Step 2: 近隣のスペースをカテゴリ別カラーピンで表示する
          🟡 カフェ  🔴 ファストフード  🟢 レンタルスペース  ⚫ コワーキング  🔵 公共施設
  Step 3: ピンをタップしてスペースの詳細を表示する
  Step 4: 詳細画面からレビュー・予約・経路案内を確認する

[ 条件検索 ]
  ├── 用途（もくもく作業 / ミーティング / 勉強 / リラックス）
  ├── 時間（利用開始〜終了時間でフィルタリング）
  └── 料金（無料 / ワンドリンク制 / 有料）
```

## 12. リスク・課題

| リスク | 影響度 | 対応策 |
|--------|--------|--------|
| スペース情報の鮮度・精度 | 高 | UGCによる情報更新促進、定期クローリング |
| 競合サービス（Googleマップ等）との差別化不足 | 高 | 「作業目的特化」の絞り込み精度で差別化を継続 |
| ユーザーのプライバシー懸念（位置情報） | 中 | 位置情報の利用説明の明確化、オプトアウト対応 |
| APIコスト（Google Maps等）の高騰 | 中 | キャッシュ戦略・リクエスト最適化、代替APIの検討 |
| ユーザー獲得コスト | 中 | SNSでの口コミ・フリーランスコミュニティへのPR活動 |
